## Decisions Made

1. **`@lru_cache(maxsize=None)` kept with `# noqa: UP033`** — ruff preferred `@cache` (Python 3.9+ stdlib alias) but the plan's acceptance criteria explicitly grep for `@lru_cache(maxsize=None)`, and the loader test's `load_reference.cache_clear()` relies on the explicit `lru_cache` wrapper API. The `# noqa` documents the intentional preservation of the explicit form.

2. **Mypy overrides for `yaml` + `dateutil.*` instead of `types-PyYAML`/`types-python-dateutil` stub packages** — Mirrors the existing `numpy_financial` precedent (`pyproject.toml:57-59`). Rationale: keeps dev-deps minimal; PyYAML and python-dateutil are stable enough that runtime introspection is sufficient. If we hit a yaml-specific type bug later, we'll add the stubs.

3. **`County`/`LoanType` moved into `TYPE_CHECKING` block in `lib/rules/loan_type.py`** — ruff TC001 fired because both are used only in function annotations (not at runtime). Test files instantiate `County(...)` at runtime via direct import. The plan's `key_links` regex `from lib.rules.types import` still matches because the import line is preserved (just guarded by `if TYPE_CHECKING:`).

4. **`Sacramento CA` and `Solano CA` listed at baseline ($832,750)** — confirms the `high_cost_counties` table includes equal-to-baseline entries (these counties' published one_unit limit IS the baseline). This matters because downstream queries iterating the table need a complete picture, not just true-overrides. Future Phase 2 county tables (FHA REF-02, VA REF-04) should follow the same convention.

5. **`Property.original_value: Money`** (not redefined locally) — D-PHASE2-Q4 + PATTERNS line 159 mandate "DO NOT redefine `Money` or `Rate`." Imported from `lib.models` with `# noqa: TC001` because Pydantic resolves annotations at runtime.

## Next Phase Readiness

### Conventions established for plans 02-02..02-06 to inherit (load-bearing!)

1. **Predicate file template:** module docstring with `Citation:` / `Source URL:` / `Effective:` strings + http(s) URL anywhere in docstring (citation-coverage meta-test enforces structurally; failure messages name RUL-12).
2. **Reference YAML template:** top-level `source:` URL string + `effective:` ISO-8601 unquoted date + `notes:` description; body uses regulator-specific keys; numeric scalars QUOTED strings (Pitfall 1).
3. **Fixture filename convention:** `tests/fixtures/rules/{predicate_stem}_{descriptive_slug}.json` (one fixture per file); mandatory `citation` + `source_url` + `comment` fields. The citation-coverage test verifies ≥1 file matches each predicate stem.
4. **Loader idiom:** `from lib.rules._loader import load_reference; ref = load_reference("YAML-stem")`. Wrap money in `Decimal(str_value)` at consumption.
5. **Cross-plan stub idiom:** When predicate B depends on predicate A's reference YAML that hasn't shipped yet, raise `NotImplementedError` with a message containing the blocking plan ID (e.g., `"02-02"`). Add a positive test asserting the stub fires; the wiring plan rewrites it.
6. **Fail-loud-on-missing-data:** When predicate cannot answer because reference data is incomplete, raise a custom exception (subclass of `ValueError` for callers; subclass of `KeyError` for loader-side missing-field errors). Never silently default. Adopt the cfpb/jumbo-mortgage `needCounty` sentinel pattern.
7. **TYPE_CHECKING discipline:** types from `lib/rules/types.py` used only in annotations should be inside `if TYPE_CHECKING:` to satisfy ruff TC001; pydantic-resolved annotations (e.g., Money on a BaseModel) need `# noqa: TC001` because Pydantic introspects annotations at runtime.
