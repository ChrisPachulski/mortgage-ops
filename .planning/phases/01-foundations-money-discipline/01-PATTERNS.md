### `lib/money.py` (source, library)

**Pattern source:** RESEARCH.md Pattern 1 (lines 244–275).
**Sibling analog:** NONE. Card-ops uses float for rate arithmetic; that is exactly the gap Phase 1 closes.
**Excerpt to copy (RESEARCH.md lines 244–275):**
```python
from decimal import Decimal, ROUND_HALF_UP, getcontext, localcontext, Context
from typing import Final

CENT: Final[Decimal] = Decimal("0.01")
MONEY_CONTEXT: Final[Context] = Context(prec=28, rounding=ROUND_HALF_UP)

def to_money(value: str) -> Decimal:
    return Decimal(value)

def quantize_cents(value: Decimal) -> Decimal:
    with localcontext(MONEY_CONTEXT):
        return value.quantize(CENT, rounding=ROUND_HALF_UP)
```
**Conventions established:** `localcontext(MONEY_CONTEXT)` is the *only* legal way to quantize; module-level `getcontext()` mutation is forbidden (Pitfall 9).

## Shared Patterns

### Decimal Discipline (applies to: `lib/money.py`, `lib/models.py`, all tests, `golden_pmt.json`)

**Source:** CLAUDE.md "Money discipline" section + RESEARCH.md Pattern 1 + Pitfalls 1, 2, 9.
**Concrete rule:**
- Construct: `Decimal("0.065")`, never `Decimal(0.065)`
- Quantize: `quantize_cents(value)` (which uses `localcontext(MONEY_CONTEXT)` + `ROUND_HALF_UP`); never bare `value.quantize(Decimal("0.01"))` (defaults to banker's rounding)
- Compare: exact equality (`assert x == Decimal("1264.14")`), never `pytest.approx`
- Serialize: Pydantic's default JSON-string output is correct; document it in DATA_CONTRACT.md, do not "fix" it (Pitfall 3)

**Source:** RESEARCH.md Pattern 2 + Pitfall 10.

### Pre-commit + CI Gate Symmetry (applies to: `.pre-commit-config.yaml`, `.github/workflows/ci.yml`)

**Source:** RESEARCH.md Patterns 5, 6 + Pitfalls 5, 6.
**Concrete rule:**

- mypy in pre-commit needs `additional_dependencies: [pydantic>=2.13.3, python-dateutil>=2.9.0]`; mypy in CI gets them via `uv sync --locked --dev`
- CI uses `uv sync --locked` (not plain `sync`) so a forgotten `uv.lock` commit fails the build loud

## Conventions to Establish

Phase 1 sets these conventions for the rest of the project. Later phases inherit them; the planner should call them out as load-bearing decisions in the per-task plan.

1. **Decimal helpers location:** `lib/money.py` exports `to_money`, `quantize_cents`, `CENT`, `MONEY_CONTEXT`. Every later module imports from here; nobody constructs `Decimal` from a literal in scattered places.
2. **Pydantic field shape:** `Annotated[Decimal, Field(strict=True, max_digits=14, decimal_places=2, ge=Decimal("0"))]` is the `Money` type alias; `Annotated[Decimal, Field(strict=True, max_digits=7, decimal_places=6, ge=Decimal("0"), le=Decimal("1"))]` is `Rate`. Phase 4+ models import these aliases from `lib/models.py`.
3. **JSON-fixture schema:** Every test fixture file lives under `tests/fixtures/{name}.json` with the structure `{"fixtures": [{"id": "...", "source": "...", ...immutable fields..., "notes": "..."}]}`. All numeric values are JSON strings (Decimal-safe). Loaded via `tests/conftest.py::golden_fixture(fixture_id)`.
4. **CI workflow file location:** `.github/workflows/ci.yml` (single workflow, single job named `check`). Future phases adding workflows (e.g. nightly regulatory-data refresh) get sibling `.yml` files in the same directory; never edit `ci.yml`'s scope.
5. **Pre-commit pin discipline:** `rev:` versions in `.pre-commit-config.yaml` are exact pins matching the dev-deps versions in `pyproject.toml`. Bumping ruff means bumping both files in the same commit.
6. **Layer naming:** User / System / Data / Reference. The layer names appear identically in `DATA_CONTRACT.md`, `.gitignore` section comments, and `scripts/hooks/block-user-layer.py` source comments. No synonyms.
7. **`.example.yml` skeleton convention:** Any user-layer YAML ships a committed `.example.yml` skeleton with redacted/dummy values; the real file is gitignored and never committed. (Inherited from card-ops shape.)
8. **`.gitkeep` whitelist convention:** Empty seam directories use `.gitkeep`, listed both in `.gitignore` (as `!path/.gitkeep`) and the user-layer hook's `ALLOWED_KEEP_FILES`. Both lists must stay in sync.
9. **Wave ordering as merge boundary:** Each wave's files must land together (W1, W2, W3 are commit boundaries, not just task-list groupings). The phase gate (`uv run ruff check . && uv run ruff format --check . && uv run mypy --strict . && uv run pytest`) must pass at each wave boundary, not only at phase end.
