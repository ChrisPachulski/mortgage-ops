## 1. HTTP API (canonical path per D-12-LIVE01-01)

Phase 12 ships `scripts/fred_cli.py` as the **canonical** FRED integration path.
The HTTP wrapper calls the FRED `series_observations` endpoint directly via
stdlib `urllib.request` — no extra runtime dependencies.

**Endpoint:**
```
GET https://api.stlouisfed.org/fred/series/observations?series_id={SID}&api_key={KEY}&file_type=json&sort_order=desc&limit=1
```

**Series allowlist (V5 input validation):**
- `MORTGAGE30US` — 30-yr fixed conventional, weekly (Thursday noon ET).
- `MORTGAGE15US` — 15-yr fixed conventional, weekly (Thursday noon ET).

Both series source from Freddie Mac's Primary Mortgage Market Survey (PMMS).

**Auth:** `FRED_API_KEY` environment variable. Register a free key at
https://fred.stlouisfed.org/docs/api/api_key.html. The key is 32-char
alphanumeric; never commit it.

**Rate limits:** 120 requests / minute. We never hit this — cache-first
read-through means the typical 7-day window has at most 1 network call per series.

**Output JSON shape (single-line stdout):**
```json
{
  "series_id": "MORTGAGE30US",
  "value": "6.84",
  "observation_date": "2026-04-25",
  "fetched_at": "2026-04-26T17:00:03Z",
  "source_url": "https://api.stlouisfed.org/fred/series/observations?series_id=MORTGAGE30US&api_key=***&file_type=json&sort_order=desc&limit=1",
  "fred_realtime_start": "2026-04-26",
  "fred_realtime_end": "2026-04-26",
  "error": null
}
```

**Always exits 0** — failures (missing FRED_API_KEY, network error, FRED schema
drift) replace `value`/`observation_date` with `null` and populate the `error`
field. This is the SKILL.md recovery contract per D-12-LIVE02-01: read the
envelope's `error` field, never depend on a non-zero exit code.

## 3. Cache Schema

`lib/fred_cache.py` writes per-series cache files at `data/cache/fred_{series_id}.json`.

**File schema (PINNED by 12-RESEARCH.md §Pattern 2):**
```json
{
  "schema_version": 1,
  "entries": {
    "MORTGAGE30US": {
      "value": "6.84",
      "observation_date": "2026-04-25",
      "fetched_at": "2026-04-26T17:00:03Z",
      "source_url": "https://api.stlouisfed.org/fred/series/observations?series_id=MORTGAGE30US&api_key=***&file_type=json&sort_order=desc&limit=1",
      "fred_realtime_start": "2026-04-26",
      "fred_realtime_end": "2026-04-26",
      "error": null
    }
  }
}
```

Field semantics:
- **`value`** — JSON STRING per CLAUDE.md money discipline (D-19 inherited).
  Downstream consumers (SKILL.md cite, evals/runner.py) Decimal-parse via
  `Decimal(str(...))`. NEVER `float()`.
- **`observation_date`** — FRED observation date (typically the survey Thursday).
- **`fetched_at`** — OUR fetch timestamp, ISO-8601 UTC with `Z` suffix. The
  7-day TTL compares against this.
- **`source_url`** — Audit trail of what we hit. `api_key=***` ALWAYS redacted
  (Pitfall 6 mitigation; the real key never lands on disk).
- **`fred_realtime_start/end`** — FRED's "as of when did this value exist"
  fields; useful to detect upstream revisions.
- **`error`** — null on success; populated string on fetch failure (never raises).

**TTL semantics (strict `<` boundary per D-12-LIVE02-01 + 12-RESEARCH.md §Pitfall 2):**
- 6d 23h 59m old → FRESH (no refetch).
- 7d 0h 0s old → STALE (refetch).
- 8d old → STALE (refetch).

FRED publishes Thursdays at noon ET (Wednesday on holiday weeks). A cache
entry fetched Thursday at 12:00:01 will be exactly 7 days "old" the following
Thursday at 12:00:01 — our strict-`<` convention means that re-invocation
triggers a refetch (catches the latest weekly print).

**Lock semantics (Python port of `orchestration/lockfile.mjs`):**
- Lock file: `data/cache/.fred-cache.lock` (gitignored).
- JSON shape: `{"pid": int, "acquired_at": int_ms_epoch, "reason": str}`.
- Stale detection: `acquired_at` > 60 seconds before `now()` → overwrite.
- NOT `O_EXCL` — read-back-and-verify CAS instead (poor-man's atomic CAS).
- Concurrent acquire: poll every 100ms up to 30s; raise `FredCacheLockError`.

## 6. Pitfalls (verbatim from 12-RESEARCH.md, applied)

**Pitfall 1: SKILL.md `!`...`` injection silently fails when FRED_API_KEY is
unset.** Mitigation: `scripts/fred_cli.py` ALWAYS exits 0 with a JSON envelope;
SKILL.md prose around the cache citation tells Claude what to do when the
envelope's `error` field is non-null.

**Pitfall 2: Cache TTL boundary off-by-one.** Mitigation: strict `<` (RESEARCH
§Pitfall 2); 4 freezegun boundary tests at 6d23h59m / 7d / 8d / lock-acquire.

**Pitfall 3: Eval "passes" because the model parroted user-supplied numbers.**
Mitigation: D-12-SC3-01 STDOUT-only sourcing; score_route_match Pitfall #2b
cross-check (numeric output + no subprocess → fail BOTH gates).

**Pitfall 4: Replay transcript drift after a SKILL.md edit.** Mitigation: v1
ships `replay-stub` mode that synthesizes transcripts from prompt + oracle
(no recorded transcript files); live-mode driver is Phase 13+.

**Pitfall 5: 95% threshold is meaningless with too few prompts.** Mitigation:
22 prompts in v1 (13 anchored + 9 TBD-skip via D-12-SC4-01 three-bucket gate);
13/(13+0) = 100% with 1 failure margin = 12/(12+1) = 92.3% < 95%.

**Pitfall 6: Transcript files leak the FRED_API_KEY.** Mitigation: all
`source_url` fields ALWAYS use `api_key=***` redaction at construction time;
the real key never str-interpolates into any output channel.
