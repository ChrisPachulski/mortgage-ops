    ```python
    #!/usr/bin/env python3
    """scripts/amortize.py — JSON-in / JSON-out CLI for the amortization engine.

    Per AMRT-06 + D-17/D-18/D-19:
      - File-based input only: --input <path> (no stdin in v1, per D-18)
      - JSON output to stdout (pipe-friendly)
      - --help works without importing heavy deps (lazy-import of lib.amortize,
        numpy_financial, dateutil per D-18)
      - Pydantic v2 strict-mode validation at the boundary (D-19): float in any
        Money/Rate field is rejected immediately with a structured JSON error.

    Default biweekly_mode = "true" when frequency is "biweekly" without an explicit
    mode (D-02). The model preserves "what the caller provided"; the engine applies
    the default at schedule-generation time.

    D-17: Phase 3 keeps this script at project root. Phase 10 physically relocates
    it to .claude/skills/mortgage-ops/scripts/amortize.py — only the path moves;
    test SCRIPT_PATH constants and SKILL.md routing absorb the change.

    Input JSON shape (full):
      {
        "loan": {
          "principal": "400000.00",          // string; Pydantic strict mode rejects floats
          "annual_rate": "0.065000",         // string
          "term_months": 360,                // int
          "origination_date": "2026-05-01",  // ISO date OR omit (engine synthesizes today)
          "loan_type": "fixed"               // optional; defaults to "fixed"
        },
        "frequency": "monthly" | "biweekly", // optional; defaults to "monthly"
        "biweekly_mode": "true" | "half-monthly" | null,  // optional; null when frequency=monthly
        "extra_principal": [                 // optional; defaults to []
          {"period": 60, "amount": "5000.00", "recurring": false}
        ]
      }
    """

    from __future__ import annotations

    import argparse
    import json
    import sys
    from pathlib import Path


    def main() -> int:
        parser = argparse.ArgumentParser(
            prog="amortize",
            description="Generate an amortization schedule from a JSON loan input.",
            epilog=(
                "Input JSON shape: a Loan object plus optional 'frequency' "
                '("monthly"|"biweekly"; default "monthly"), "biweekly_mode" '
                '("true"|"half-monthly"; default "true" when frequency=biweekly), '
                'and "extra_principal" (list of {period, amount, recurring} entries). '
                "All money/rate fields MUST be JSON strings (e.g. \"400000.00\"); "
                "Pydantic v2 strict mode rejects JSON floats at the boundary."
            ),
        )
        parser.add_argument(
            "--input",
            required=True,
            type=Path,
            help="Path to JSON file containing the loan input.",
        )
        args = parser.parse_args()

        # Lazy-import after argparse (D-18: --help is fast; numpy_financial + dateutil
        # are NOT imported when the user runs `--help`).
        from pydantic import ValidationError  # noqa: PLC0415  # lazy-import per D-18
        from lib.amortize import AmortizeRequest, build_schedule  # noqa: PLC0415  # lazy-import per D-18

        try:
            raw = args.input.read_text()
        except FileNotFoundError as e:
            print(
                json.dumps({"error": f"input file not found: {e.filename}"}),
                file=sys.stderr,
            )
            return 2
        except OSError as e:
            print(
                json.dumps({"error": f"could not read input file: {e}"}),
                file=sys.stderr,
            )
            return 2

        # D-19: Pydantic v2 strict-mode validation at the script boundary.
        # Floats in any Money/Rate field are rejected here with a structured JSON error.
        try:
            request = AmortizeRequest.model_validate_json(raw)
        except ValidationError as e:
            # Pydantic emits structured JSON-readable errors; pass through as JSON.
            print(e.json(), file=sys.stderr)
            return 2

        schedule = build_schedule(
            request.loan,
            frequency=request.frequency,
            biweekly_mode=request.biweekly_mode,
            extra_principal=request.extra_principal,
        )
        print(schedule.model_dump_json(indent=2))
        return 0


    if __name__ == "__main__":
        sys.exit(main())
    ```

    Notes on the implementation:

    - **`# noqa: PLC0415` comments**: ruff's PLC0415 rule (`import-outside-top-level`) would flag the lazy-imports. The locked decision D-18 REQUIRES lazy-import; the noqa with the inline `# lazy-import per D-18` comment makes the rationale explicit and grep-able. If `pyproject.toml` does not currently enable PLC0415 (check `[tool.ruff.lint] select`), drop the noqa to avoid Rule-1 deviation; in this project's ruff config, the active selectors are `["E", "F", "W", "I", "UP", "B", "SIM", "RUF", "TCH", "PT"]` — PLC is NOT selected, so PLC0415 will not fire and the noqa would itself trip RUF100 (unused noqa). **Therefore: omit the `# noqa: PLC0415` comment entirely** since the active ruff selectors don't enable PLC. Replace with `# lazy-import per D-18` plain comments only.

    - **Use `AmortizeRequest.model_validate_json` (NOT `Loan.model_validate_json`)**: the input JSON is a top-level request object containing `loan`, `frequency`, `biweekly_mode`, `extra_principal`. AmortizeRequest captures this whole shape; using `Loan.model_validate_json` would force the loan + the request fields into one model and break the D-02 validator. PATTERNS.md line 280 + RESEARCH §7 line 538 both confirm AmortizeRequest is the correct boundary.

    - **`type=Path` for `--input`**: avoids manual `Path(args.input)` construction; argparse handles the conversion + Path's exception types are clean for the FileNotFoundError handler.

    - **Why `OSError` after `FileNotFoundError`**: `read_text()` can raise either; FileNotFoundError is the common case (typo'd filename), OSError catches permission errors / I/O failures with a slightly different message. Both exit 2 with structured JSON.

    - **Do NOT** add a `--frequency` or `--biweekly-mode` CLI flag. CONTEXT.md `<deferred>` line 199 deferred stdin; the locked D-18 says "--input <path> only". All scenario knobs live IN the JSON.

    Anti-patterns to avoid (per PATTERNS.md "scripts/amortize.py" anti-patterns):

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User filesystem (`--input` path) → script | Untrusted input (whatever JSON the user/Claude crafts). The threat surface for the entire phase. |
| JSON text → AmortizeRequest.model_validate_json | Pydantic v2 strict mode is the validation gate; it rejects floats in money fields, rejects unknown keys (extra="forbid"), and runs the D-02 cross-field validator. |
| AmortizeRequest → build_schedule | Internal trust boundary — AmortizeRequest is already validated; build_schedule trusts its inputs and produces a Schedule. |
| Schedule → stdout JSON | Schedule.model_dump_json — Pydantic-managed serialization; no user-controlled formatting. |
